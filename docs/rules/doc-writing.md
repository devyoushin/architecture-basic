# 문서 작성 규칙 (Doc Writing Rules)

Claude가 이 저장소에서 문서를 작성하거나 수정할 때 반드시 따라야 할 규칙입니다.

---

## 1. 언어 및 문체

- **기본 언어**: 한국어
- **영어 원문 병기**: 기술 용어 첫 등장 시 원문 병기
  - 예: 서비스 제어 정책 (Service Control Policy, SCP)
- **문체**: 명사형 종결 (`~함`, `~사용`, `~설정`)
- **존댓말 금지**: 경어체 사용하지 않음

## 2. 코드 블록

- **언어 태그 필수**:
  - ` ```hcl ` — Terraform
  - ` ```json ` — SCP, IAM Policy
  - ` ```bash ` — AWS CLI
  - ` ```yaml ` — CloudFormation, EKS
- **실행 가능한 코드**: 복붙해서 바로 실행/적용 가능한 수준
- **플레이스홀더**: `<ACCOUNT_ID>`, `<OU_ID>`, `<REGION>` 형식

## 3. 섹션 구성 필수 요소

| 섹션 | 내용 | 생략 가능 여부 |
|------|------|----------------|
| 1. 개요 | 왜 필요한지 | 불가 |
| 2. 설계 원칙 | 핵심 원칙 | 불가 |
| 3. 아키텍처 설계 | 구조, Terraform, 정책 | 불가 |
| 4. 운영 고려사항 | 비용, 모니터링, 주의 | 불가 |
| 5. 트러블슈팅 | 증상 → 원인 → 해결 | 가능 |
| 6. 체크리스트 | 구현 완료 기준 | 불가 |

## 4. 아키텍처 다이어그램 표현

텍스트 기반 다이어그램:
```
Management 계정
    │
    ├── Security OU
    │   ├── Log Archive 계정
    │   └── Security Tooling 계정
    │
    └── Workload OU
        ├── Dev 계정
        └── Prod 계정
```

## 5. 금지 사항

- 실제 계정 ID (12자리 숫자) 노출 금지
- 실제 ARN 노출 금지
- 하드코딩된 자격증명 금지
- 추측성 비용 수치 단독 기재 금지
