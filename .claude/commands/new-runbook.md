`$ARGUMENTS`를 파싱하여 AWS 아키텍처 운영 Runbook을 생성합니다.
- 첫 번째 인자: 대상 서비스/컴포넌트 (landing-zone, iam, network, eks 등)
- 나머지 인자: 작업명

## 파일 생성 규칙
1. 파일명: `runbook-{서비스}-{작업명}.md`
2. 저장 위치: 프로젝트 루트
3. `templates/runbook.md` 템플릿 기반으로 작성

## 작성 시 필수 포함 사항
- **사전 체크리스트**: 최소 4개 항목
- **영향 계정 목록**: 어떤 AWS 계정이 영향받는지
- **단계별 AWS CLI/Console 절차**: 복붙 즉시 실행 가능
- **확인 명령어**: 각 스텝 완료 후 검증 방법
- **롤백 절차**: 원래 상태로 복구하는 방법
- **비용 영향**: 예상 추가/절감 비용
