아래 기준으로 `$ARGUMENTS` 문서를 검토하고 개선점을 제안해 주세요.

## 검토 항목

### 1. 설계 원칙 준수
- [ ] 최소 권한 원칙이 반영됐는지
- [ ] 환경 격리 원칙이 고려됐는지
- [ ] 네트워크 중앙화 전략이 일관성 있는지

### 2. 보안 검토 (`rules/security-checklist.md` 기준)
- [ ] IAM 최소 권한 예시인지
- [ ] SCP 가드레일이 포함됐는지
- [ ] 실제 계정 ID/ARN 노출 없는지

### 3. Terraform 코드 품질
- [ ] 태그 전략이 `15-tagging.md`와 일관성 있는지
- [ ] 민감 정보 하드코딩 없는지

### 4. 운영 고려사항
- [ ] 비용 영향이 언급됐는지
- [ ] 모니터링/알람이 포함됐는지

## 출력 형식

```
## 검토 결과: {파일명}

### 잘된 점
- ...

### 개선 필요 사항
1. [섹션명] 문제 → 개선 방안

### 보안 이슈
- 위험도: High/Medium/Low
- 내용: ...
```
